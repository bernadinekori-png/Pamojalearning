# Report Portal Backend (Ready-to-run)

## Quick start

1. Copy this project folder to your machine or download the zip.
2. Create a `.env` file in the project root (see `.env.example`).
3. Install dependencies:
   ```
   npm install
   ```
4. Start MongoDB (local or Atlas) and update `MONGO_URI` in `.env`.
5. Start the server:
   ```
   npm run dev
   ```
6. Open your frontend and point fetch calls to `http://localhost:5000/api/...`

## Endpoints (high level)
- `POST /api/auth/register` - register new user
- `POST /api/auth/login` - login
- `POST /api/reports` - create report (auth required)
- `GET /api/reports/me` - get user's reports
- `GET /api/admin/reports` - admin reports (admin only)
- `PATCH /api/admin/reports/:id/status` - update report status (admin)
- `GET /api/admin/analytics` - analytics summary (admin)
- `GET /api/superadmin/users` - list users (superadmin)
- `PATCH /api/superadmin/users/:id/role` - update user role (superadmin)
- `POST /api/superadmin/broadcast` - send broadcast notification (superadmin)

## Notes
- Files uploaded via `/api/reports` are stored in `/uploads`.
- For production replace local storage with S3 and secure tokens with cookies.
